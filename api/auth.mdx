---
title: 'Authentication endpoints'
description: 'Endpoints for logging in and registering a new Underground Fleet account.'
---

Use these endpoints to authenticate users and create new accounts. Both endpoints are public — no token is required.

---

## POST /auth/login

Authenticate with a username and password. Returns a JWT token you use to access protected endpoints.

### Request body

<ParamField path="body.username" type="string" required>
  The account username.
</ParamField>

<ParamField path="body.password" type="string" required>
  The account password.
</ParamField>

### Response

<ResponseField name="token" type="string">
  A signed JWT token. Include this value in the `Authorization: Bearer <token>` header for all protected requests.
</ResponseField>

### Example

```bash
curl --request POST \
  --url http://localhost:8080/auth/login \
  --header 'Content-Type: application/json' \
  --data '{
    "username": "jsmith",
    "password": "secret123"
  }'
```

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

<Tip>
  Store the returned token in `localStorage` so the frontend can attach it automatically to subsequent requests.
</Tip>

---

## POST /auth/register

Create a new user account. On success, returns a JWT token and the user is signed in immediately.

### Request body

<ParamField path="body.username" type="string" required>
  The desired username for the new account.
</ParamField>

<ParamField path="body.email" type="string" required>
  The email address for the new account.
</ParamField>

<ParamField path="body.password" type="string" required>
  The password for the new account.
</ParamField>

### Response

<ResponseField name="token" type="string">
  A signed JWT token. The frontend stores this in `localStorage` and redirects to `/vehicles` immediately after successful registration.
</ResponseField>

### Example

```bash
curl --request POST \
  --url http://localhost:8080/auth/register \
  --header 'Content-Type: application/json' \
  --data '{
    "username": "jsmith",
    "email": "jsmith@example.com",
    "password": "secret123"
  }'
```

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

<Note>
  Registration returns a token directly. The frontend stores it in `localStorage` and redirects to `/vehicles` — no separate login step is needed.
</Note>
