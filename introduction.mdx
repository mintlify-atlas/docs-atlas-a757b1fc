Underground Fleet is a React-based vehicle reservation SaaS application. It lets users browse a catalog of available vehicles, create and manage reservations, and complete online check-in — all from a browser.

The frontend connects to a REST API backend running at `http://localhost:8080`. Authentication uses JWT tokens stored in `localStorage`.

## Key features

- **Vehicle catalog** — GET `/vehicles` returns a list of vehicles rendered as cards. No authentication required to browse.
- **Reservations** — POST `/reservations` creates a booking. GET `/reservations/mine` returns your personal reservation list. This route is protected and requires a valid JWT.
- **Online check-in** — Submit your last name, IRN reservation code, and pickup date via the `/checkin` page.
- **JWT authentication** — Register at `/register` or sign in at `/login`. The token is stored in `localStorage` and sent as a bearer token on all API requests.

| Backend API | REST, `http://localhost:8080` |

## Architecture

The frontend is a single-page application built with Vite. It communicates exclusively with a REST API at `http://localhost:8080`. The Axios client in `src/api/axiosConfig.js` attaches the JWT bearer token from `localStorage` to every request automatically.

Protected routes (such as `/reservations`) use a `PrivateRoute` component that redirects unauthenticated users to `/login`.

<Note>
  You must have the backend API running at `http://localhost:8080` before the frontend can authenticate users, load vehicles, or process reservations.
</Note>
